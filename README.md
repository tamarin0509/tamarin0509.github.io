# RSI Breakout Analyzer & Auto Screener

RSI（相対力指数）のトレンドライン・ブレイクアウト手法を用いたテクニカル分析チャート、および為替・株式を毎日一括スキャンする自動スクリーナーツールです。

GitHub Actionsを利用したサーバーレスバックエンドによる自動最適化スクリーニングと、インタラクティブなTradingView風チャートUIを統合した高機能なシステムになっています。

---

## 🚀 主な機能

### 1. RSIトレンドライン ＆ ブレイクアウト検知
- **極大・極小点（山・谷）の自動検出**:
  - **Kairiゼロ交差方式**: RSIとRSI移動平均線（MA）のクロス（ゴールデン/デッド）の間の「波」の中で最高値・最安値を極値とします。
  - **スイングピボット方式**: 基準足の前後スイング期間（左右10本など）の中で最高・最低となる山・谷を厳密に検出します。
- **ブレイクアウト判定**:
  過去の直近極値から描かれた抵抗線・支持線（予想線）をRSIがマージンを超えて突破した瞬間に「BUY」「SELL」のサインを点灯します。

### 2. バックテスト検証（リペイント検証）
- **Walk-Forward Replay**:
  チャートを後から一括計算した「後知恵の線」と、過去にタイムスリップして1日ずつ進めて計算した「実戦の線」を比較検証します。
  - `確定 [確]`: リアルタイムでも発生し、現在もチャートに残っている信頼性の高いシグナル。
  - `消滅 [消]`: リアルタイムで一度画面に出たものの、その後の波形変化（リペイント）でトレンドラインが引き直されて歴史から消えたダマシシグナル。
  - `実戦のみ [実]`: 当時の波形でブレイクしたが、最新のライン描画制限などで過去チャートから見えなくなっているシグナル。

### 3. マルチタイムフレーム（MTF）フィルター
- 日足チャートの中に「週足（上位足）の確定RSIトレンド」を統合。
- 週足RSIが強気（50以上）の時は買い（BUY）のみ、弱気（50未満）の時は売り（SELL）のみに取引を制限することで、上位トレンドに逆らう無駄なダマシトレードを強力に抑制します。

### 4. 銘柄別パラメータ最適化（オートフィッティング）
- 過去2年間のヒストリカルデータに対し、RSI期間・MA期間・判定マージン・オフセットの最適な組み合わせを高速にグリッドサーチ。
- 15営業日後の期待リターンが最大になるパラメータを自動決定します。
- **個別チャート上での手動最適化**、および**スクリーナー内の自動最適化**の両方に対応。

### 5. 自動スクリーニング・バックエンド (GitHub Actions)
- 毎日午前6:00 (日本時間) にGitHubサーバー上で自動的にスクリーナーが実行。
- 全66銘柄（主要為替、日米株式12セクター、主要4指数、暗号資産）をスキャンし、Kairi/Swingのどちらのアルゴリズムが適しているかも含めて最適パラメータを自動算出。
- 最新サイン発動銘柄の一覧を `screener_results.json` に出力し、フロントエンドに完全自動連動させます。

### 6. セクターマップ — Market Galaxy（RSI軌道マップ）
- 市場全体を「銀河」として俯瞰する独自の極座標ビジュアライゼーション。**銘柄＝惑星、セクター＝星域（扇形）、中心からの距離＝RSI**。
- 売られすぎ（RSI低）の星は中心へ落ち込んで青く冷え、買われすぎ（RSI高）の星は外縁で赤く燃える発散カラースケール。星の大きさは前日比変動率。
- ブレイクアウトシグナル発生中の銘柄はパルサーのように明滅。ホバーで詳細ツールチップ、クリックで最適パラメータ適用済みの個別チャートへジャンプ。
- 従来型のセクター別タイルグリッド（ヒートマップ）へもワンクリックで切り替え可能。

## 🛠 構成ファイル

- `index.html`: ダッシュボードUI (個別チャート分析、自動スクリーナー、セクターマップ、バージョン履歴のタブ切り替え)
- `rsi_breakout.js`: フロントエンドの描画・UIイベント処理・ローカル最適化ロジック
- `style.css`: ガラスモルフィズムやダークテーマを採用したモダンなスタイルシート
- `scripts/screener.js`: バックエンド用スクリーニング・最適化Node.jsスクリプト
- `.github/workflows/screener.yml`: GitHub Actions 定期実行用設定ファイル
- `screener_results.json`: スクリーナーの計算結果データ (Actionsによって自動更新)

## 💻 使い方・開発

### ローカルでのスクリーナー手動実行
Node.js (18以降推奨) がインストールされている環境で、以下のコマンドを実行すると、ローカルで全銘柄のスクリーニングと最適化を即座に実行し、`screener_results.json` を更新できます。

```bash
node scripts/screener.js
```

### ローカルでのサーバー起動
静的HTMLファイル群であるため、特別なビルド手順は不要です。CORSなどの制約を避けるため、簡易ローカルサーバーを使って動作させることをお勧めします。

```bash
# Pythonの場合
python -m http.server 8000

# Node.js (http-server) の場合
npx http-server -p 8000
```
起動後、ブラウザで `http://localhost:8000` にアクセスしてください。
