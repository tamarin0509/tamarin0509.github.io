# RSI Breakout Analyzer：仕様書 ＆ ユーザー利用マニュアル

RSIブレイクアウト分析ツール「RSI Breakout Analyzer」のシステム仕様、アルゴリズムの仕組み、およびユーザー向け操作方法を記述した公式マニュアルです。

---

## 1. システム概要と特徴

本ツールは、価格ではなくオシレーターである**RSI（相対力指数）**の上にトレンドライン（上値抵抗線・下値支持線）を引き、そのラインをRSIが上下に突き抜けた瞬間（ブレイクアウト）を売買シグナルとして検知する高度テクニカル分析システムです。

### 🌟 コア機能
1. **個別チャート分析**: リアルタイムで任意のティッカーシンボル（為替・日本株・米国株・暗号資産）を検索し、RSIトレンドラインと売買サイン（BUY / SELL）を描画します。
2. **自動スクリーナー**: 主要66銘柄（日米株12セクター・為替・指数・暗号資産）を常に監視し、最新シグナル（直近15日以内）が発動している銘柄を毎日自動でリストアップします。
3. **セクターマップ "Market Galaxy"**: 市場を銀河として俯瞰する極座標ビュー。銘柄＝惑星、セクター＝星域、中心からの距離＝RSI。シグナル発生中の星は明滅します。従来型のタイルヒートマップにも切替可能。
4. **パラメータ自動最適化（WFO）**: 過去2年間のデータを用いて、その銘柄が最も利益を出しやすい設定を総当たり（2,940通り）でバックテストし、自動フィッティングします。
5. **GitHub Actions バックエンド**: 毎朝自動でクラウド（GitHubサーバー）上でスクリーニングと最適化が実行され、結果をプッシュします。
6. **バックテストモード（タイムスライド）**: 最大250営業日前まで過去に遡り、「当時の最新データ」だけでチャート・トレンドライン・シグナルを完全再現します。

## 2. コアアルゴリズムと仕様

### ① 2種類の極値検出（山・谷の検出）
トレンドラインを引くための「山（極大点）」と「谷（極小点）」を検出するアルゴリズムを2つ搭載しています。
* **Kairiゼロ交差（乖離率方式）**:
  RSIとその移動平均線（MA）が交差する「波」の範囲内で、最高値・最安値を極値とします。ダイナミックな波の頂点を捉えやすく、為替（FOREX）などに向いています。
* **スイングピボット（左右比較方式）**:
  指定した期間（スイング期間。例：前後10日）の中で、自身が最も高い（または低い）ポイントを極値とします。局所的な反転ポイントを厳密に検出でき、個別株式などに向いています。

### ② マルチタイムフレーム（MTF）フィルター
日足のシグナルに対し、**週足のRSIトレンド（RSI基準値50）**をフィルターとして重ねます。
* 週足RSI $\ge 50$ （強気相場）: **買い（BUY）シグナルのみ**を通し、売りはダマシとして除外。
* 週足RSI $< 50$ （弱気相場）: **売り（SELL）シグナルのみ**を通し、買いはダマシとして除外。
上位足の大きなトレンドに逆行するトレードを物理的にカットすることで、勝率を大幅に向上させます。

### ③ ウォークフォワード・パラメータ最適化
過去2年間のデータを 3:1 に分割して評価し、過去の相場だけに偶然適合した設定（過学習）を排除します。
1. **インサンプル（過去の最初の1.5年）での最適化**:
   RSI期間、MA期間、マージン、オフセットの全**2,940通り**の組み合わせについてバックテストを実行し、期待値（Expectancy）の最も高い上位5候補を選定します。
2. **アウトサンプル（直近0.5年/未知データ）での検証**:
   上位5候補を直近0.5年の未知データに当てはめ、この検証期間中での成績が最も高く、かつ損失を出していない頑健な設定を最終採用します。

## 3. ユーザー向け利用マニュアル（操作ガイド）

本システムは、ブラウザだけで完結する直感的かつプレミアムなデザインの画面構成になっています。

### 【ステップ 1：自動スクリーナーでチャンス銘柄を探す】
1. 画面上部の **「自動スクリーナー」タブ** をクリックします。
2. 監視銘柄一覧テーブルが表示され、各銘柄の「現在値」「前日比」「現在のRSI」およびバックエンドで自動最適化された「最適パラメータ」を確認できます。
3. **「最新シグナル」列** を確認します。
   * **▲ BUY**: 直近で買いブレイクアウトが発生した銘柄（カッコ内は何日前に発生したかを示します）。
   * **▼ SELL**: 直近で売りブレイクアウトが発生した銘柄。
   * `なし`: 現在有効な最新シグナルはありません。

### 【ステップ 1.5：セクターマップで市場全体を俯瞰する】
1. 画面上部の **「セクターマップ」タブ** をクリックすると、**Market Galaxy（RSI軌道マップ）**が表示されます。
2. 読み方：
   * **中心からの距離＝RSI**（内側の軌道＝売られすぎ、外側の軌道＝買われすぎ。30 / 50 / 70 のリングが目印）。
   * **星の色**：青（売られすぎ）〜グレー（中立）〜赤（買われすぎ）。
   * **星の大きさ**：前日比変動率の大きさ。
   * **明滅する星**：直近15日以内にブレイクアウトシグナル（▲BUY / ▼SELL）が発生中。
   * **中心のコア**：全銘柄の市場平均RSI。
3. 星にマウスを乗せると銘柄名・終値・前日比・RSI・シグナルの詳細ツールチップが表示され、クリックすると最適パラメータ適用済みの個別チャートへジャンプします。
4. 右上の **「タイルビュー」** ボタンで、従来型のセクター別タイルヒートマップ（RSI昇順）にも切り替えられます。

### 【ステップ 2：個別チャートでの詳細分析と検証】
1. スクリーナーテーブルの中で、シグナルが出ている銘柄の行（または右端の **「表示」** ボタン）をクリックします。
2. **「個別チャート分析」タブ**に自動で切り替わり、選択した銘柄の日足チャート（価格＋RSI）が描画されます。
3. **重要**: ジャンプした際、その銘柄用に最適化されたパラメータ（RSI期間、MA期間、マージン、オフセット）がサイドバーのスライダーと自動同期し、チャート上にトレンドラインとシグナルマーク（BUY/SELL矢印）が自動描画されます。

### 【ステップ 3：手動でのパラメータ微調整と即時最適化】
1. **スライダー操作**: 
   サイドバーにある「RSI基準値」「RSI移動平均期間」「ブレイク判定マージン」などを手動で変更すると、チャート上のラインとシグナルが即座に再計算・再描画されます。
2. **「銘柄パラメータ最適化」ボタン**:
   サイドバー上部のこのボタンをクリックすると、ブラウザ上で直ちに2,940通りのバックテストとウォークフォワード検証が実行され、現在の銘柄に最適な最新パラメータにスライダーが自動で再フィッティングされます（フリーズせず一瞬で終わります）。
3. **「チャートを更新」ボタン**:
   サイドバーの最下部にあり、最新のYahoo Financeデータを再ロードしてチャートをクリア・更新します。

### 【ステップ 4：シグナル履歴と過去検証の確認】
個別チャート分析タブの下部には、そのパラメータ設定における過去の「シグナル履歴テーブル」があります。
* **履歴の移動**: 
  テーブル右端の **「移動」** ボタンをクリックすると、該当するシグナルが発生した過去の日時へチャートの拡大表示（タイムスケール）が自動でジャンプします。当時のブレイクアウトの瞬間をビジュアルで確認できます。

### 【ステップ 5：バックテストモードで過去にタイムスライドする】
サイドバーの **「バックテストモード」** スライダーを使うと、指定した営業日数（最大250日 ≒ 約1年）だけ過去に遡り、**その時点で実際に見えていた**チャート・トレンドライン・シグナル・統計を再現できます。
1. **スライダー**: 遡る営業日数を指定します（0＝最新）。スライダーを離した時点で再計算されます。
2. **「◀ 1日戻る / 1日進む ▶」ボタン**: 1営業日ずつコマ送りできます。日々のライン引き直しやシグナルの出現・消滅（リペイント）の様子を検証するのに便利です。
3. **「リセット」ボタン**: 最新データの表示に戻ります。
* バックテスト中はチャート上部にオレンジ色のバナーで「基準日（当時の最新日付）」が表示されます。
* 未来のデータは一切参照されません。バックテスト中に「銘柄パラメータ最適化」を実行した場合も、当時のデータのみで最適化されます（未来リーク防止）。
* 銘柄・表示期間を切り替えてもオフセットは維持されるため、同じ日付時点での複数銘柄の比較が可能です。

## 4. 管理者・システム運用仕様

### ① デイリースクリーニングの自動実行
* **実行スケジュール**: 毎日日本時間 **早朝 6:00**（UTC 21:00）に GitHub Actions クラウド上でスクリプト（`screener.js`）が自動起動します。
* **プッシュサイクル**: 取得した最新の終値と最適化パラメータが計算され、結果ファイル `screener_results.json` がリポジトリへ自動コミット＆プッシュされます。数分後、Web公開されているサイトへ自動反映されます。

### ② スクリーナーの手動強制更新方法
昨日のアメリカ市場終了直後など、今すぐスクリーナーテーブルを最新化したい場合は、GitHub上から直接手動でスクリプトを実行できます。
1. あなたの GitHub リポジトリ（`tamarin0509.github.io`）にアクセスします。
2. **「Actions」** タブをクリックします。
3. 左側のワークフロー一覧から **「Auto Screener」** を選択します。
4. 右側にある **「Run workflow」** ボタンをクリックし、ブランチが `main` であることを確認して実行します。
5. 約20秒〜30秒で処理が完了し、サイトが最新データに更新されます。
